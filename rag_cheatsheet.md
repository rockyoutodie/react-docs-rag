# RAG 速查手册

> 学习时间: 2026-05-10

## 🌟 RAG 一句话定义 Retrieval-Augmented-Generation 检索增强生成

RAG = 让LLM临时学习某些特定领域并且LLM之前并不知道的内容，来回答问题

## 🗺️ RAG 完整流程

### 阶段 A: 离线建库
1. 文档解析（parser）
2. 切分（chunking）
3. 生成向量（embedding）
4. 存入向量数据库（vector DB）

### 阶段 B: 在线查询
5. 把问题也转成向量
6. 在向量数据库里搜相似的 chunks（检索 retrieval） 找到 Top-K 相关段落
7. 把“问题 + 相关段落”拼成 Prompt
8. 发给 LLM
9. LLM 基于上下文给出答案

## 🔑 关键概念

### Embedding
- 是什么: 生成向量
- 我的理解: 把一小段文字变成一串数字（向量）

### Chunk Size
- 是什么: 每小段文字的大小
- 选择原则: 固定长度、按句号切、递归切分、语意切分

### Top-K
- 是什么: 和问题向量最近的 K 个向量
- 经验值: 3-5

### 向量数据库
- 我会用的: Chroma
- 为什么不用 MySQL: MySQL 擅长“精确匹配”，而 RAG 要找“最相似”的结果，普通数据库做不到或者很慢。需要专门为此设计的向量数据库
